# Decisions Log

One-line entries capturing architectural and naming decisions made during sessions. Auto-appended by Claude Code when it picks between alternatives with non-obvious rationale — the kind of choice where a future reader would ask "why did we do it this way?"

Format: `- YYYY-MM-DD: {decision}. Reason: {why}. Alternatives considered: {optional}.`

Example:
- 2026-04-13: Use a single deploy.md entry point instead of three deployment/*.md files. Reason: mirrors merge.md and preview.md; simpler for the Deploy button to reference one path. Alternatives considered: keep the three-file structure for large projects.

_Delete the example once this file has real entries._
